# Supabase Setup Guide for Biltone Supplies

> [!IMPORTANT]
> **Fixing "Error loading products" (Safe Way)**:
> The reason you see an error is usually because the website is looking for specific column names (`title`, `image`, `stock`). If your table was created with different names (like `name` or `image_url`), it will fail.
>
> **Option 1: Safe Migration (Keep your data)**
> Run this if you already have products and don't want to delete them:
> ```sql
> -- Rename columns to match what the website expects
> ALTER TABLE products RENAME COLUMN name TO title;
> ALTER TABLE products RENAME COLUMN image_url TO image;
> ALTER TABLE products RENAME COLUMN stock_quantity TO stock;
> 
> -- Ensure security doesn't block loading
> ALTER TABLE products DISABLE ROW LEVEL SECURITY;
> ```
>
> **Option 2: Clean Start (Delete and Recreate)**
> Run this ONLY if the table is empty or you don't mind starting over:
> ```sql
> DROP TABLE IF EXISTS products;
> CREATE TABLE products (
>   id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
>   title TEXT NOT NULL,
>   description TEXT,
>   price DECIMAL(10,2) NOT NULL,
>   category TEXT,
>   image TEXT,
>   stock INTEGER DEFAULT 0,
>   offer_price DECIMAL(10,2),
>   offer_expires TIMESTAMP WITH TIME ZONE,
>   created_at TIMESTAMP WITH TIME ZONE DEFAULT TIMEZONE('utc'::text, NOW())
> );
> ALTER TABLE products DISABLE ROW LEVEL SECURITY;
> ```

## 1. Create Tables
Go to the **SQL Editor** in your Supabase Dashboard and run the following code to create all necessary tables:

```sql
-- 1. Create Profiles Table (for user roles and names)
CREATE TABLE profiles (
  id UUID REFERENCES auth.users ON DELETE CASCADE PRIMARY KEY,
  email TEXT UNIQUE,
  full_name TEXT,
  role TEXT DEFAULT 'admin',
  approved BOOLEAN DEFAULT TRUE,
  created_at TIMESTAMP WITH TIME ZONE DEFAULT TIMEZONE('utc'::text, NOW())
);

-- 2. Create Products Table
CREATE TABLE products (
  id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  title TEXT NOT NULL,
  description TEXT,
  price DECIMAL(10,2) NOT NULL,
  category TEXT,
  image TEXT,
  stock INTEGER DEFAULT 0,
  offer_price DECIMAL(10,2),
  offer_expires TIMESTAMP WITH TIME ZONE,
  created_at TIMESTAMP WITH TIME ZONE DEFAULT TIMEZONE('utc'::text, NOW())
);

-- 3. Create Orders Table
CREATE TABLE orders (
  id TEXT PRIMARY KEY,
  customer_details JSONB,
  items JSONB,
  total DECIMAL(10,2),
  status TEXT DEFAULT 'Pending',
  created_at TIMESTAMP WITH TIME ZONE DEFAULT TIMEZONE('utc'::text, NOW())
);

-- 4. Create Messages Table
CREATE TABLE messages (
  id TEXT PRIMARY KEY,
  name TEXT,
  email TEXT,
  subject TEXT,
  message TEXT,
  status TEXT DEFAULT 'New',
  created_at TIMESTAMP WITH TIME ZONE DEFAULT TIMEZONE('utc'::text, NOW())
);

-- 5. Create Content Table (for About Us, etc.)
CREATE TABLE content (
  key TEXT PRIMARY KEY,
  value JSONB,
  updated_at TIMESTAMP WITH TIME ZONE DEFAULT TIMEZONE('utc'::text, NOW())
);
```

## 2. Automate Profile Creation
Run this code to ensure that every time someone registers, a record is automatically created in the `profiles` table:

```sql
-- Create a trigger function
CREATE OR REPLACE FUNCTION public.handle_new_user()
RETURNS TRIGGER AS $$
BEGIN
  INSERT INTO public.profiles (id, email, full_name, role, approved)
  VALUES (new.id, new.email, new.raw_user_meta_data->>'full_name', 'admin', true);
  RETURN NEW;
END;
$$ LANGUAGE plpgsql SECURITY DEFINER;

-- Set the trigger
CREATE TRIGGER on_auth_user_created
  AFTER INSERT ON auth.users
  FOR EACH ROW EXECUTE PROCEDURE public.handle_new_user();
```

## 3. Row Level Security (RLS)
By default, Supabase tables are protected. For this prototype, you can disable RLS for simplicity, or add these policies:

### Disable RLS (Easiest for testing)
Run this for each table if you want everyone to be able to read/write (not recommended for production but okay for early stages):
```sql
ALTER TABLE products DISABLE ROW LEVEL SECURITY;
ALTER TABLE orders DISABLE ROW LEVEL SECURITY;
ALTER TABLE messages DISABLE ROW LEVEL SECURITY;
ALTER TABLE profiles DISABLE ROW LEVEL SECURITY;
ALTER TABLE content DISABLE ROW LEVEL SECURITY;
```

## 4. Hosting Your Website
Since you want Supabase to "host" the whole site:

1. **Option A: GitHub Pages (Recommended)**
   - Upload your files to a GitHub repository.
   - Go to **Settings > Pages** and enable GitHub Pages.
   - This is the fastest and most reliable way for static HTML sites.

2. **Option B: Supabase Storage (Workaround)**
   - You can upload your HTML/JS/CSS files to a **Public Bucket** in Supabase Storage.
   - You will get a URL like `https://...supabase.co/storage/v1/object/public/site/index.html`.
   - **Note**: This is not "true" web hosting (no custom domains easily), but it works for holding the files.

**Important**: Make sure your `js/config.js` file has the correct `SUPABASE_URL` and `SUPABASE_KEY` from your project settings.
